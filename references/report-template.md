# Report Template

每次预测使用两段式聊天框结构。第一段先输出核心 6 行赛前情报和专家席提名，等待用户确认；第二段在用户确认后输出 Paul-chartroom 专家结果和章鱼保罗判断。

不要使用大表格作为主输出。不要展开长篇网页摘要。来源只用短列表和来源索引承载，保证聊天框里可读。

## 第一段：赛前情报 + 专家提名

```markdown
# 章鱼保罗：{主队} vs {客队}

【比赛锁定】
赛事：
时间：
地点：
阶段：
本次预测生成时间：

【赛前情报｜核心 6 行】
赛程：{赛事、日期、地点、阶段，一句话}（来源：S1）
名单：{两队名单或可用阵容状态，一句话}（来源：S2）
伤停：{关键伤停/停赛/复出；没有可靠来源就明说}（来源：S3）
更衣室：{可靠团队动态；没有可靠矛盾来源就明说}（来源：S4）
状态/交锋：{近期状态 + 历史交锋合并一句}（来源：S5）
环境/变量：{主客场、旅行、海拔、天气、场地、赛程密度或权威预览关键变量}（来源：S6）

【来源索引｜只列最关键 3-5 个】
S1｜{来源名}｜{时间}｜{链接}｜{支持事实}｜可信度：高/中/低/兜底
S2｜...

【情报缺口】
- {只列真正影响判断的缺口，例如临场首发、赛前发布会未出、天气/场地信息未更新；不要列访问失败的网页清单}

【Paul-chartroom 专家席提名】
以下是基于公开方法论的模拟分析席，不代表相关人物本人实时观点或真实发言。

我建议这场开 {3-5} 个席位：

1. {专家名}｜{身份}
背景：{一句话背景}
本轮任务：{负责判断什么}
为什么选他/她：{为什么适合这场}
缺了他/她会少什么视角：{拿掉后的盲区}

2. ...

回复"开始"我就按这组专家开 Paul-chartroom。
如果你想换人，直接说"把 X 换成 Y"。
```

## 第二段：Paul-chartroom + 章鱼保罗判断

```markdown

以下是基于公开方法论的模拟分析席，不代表相关人物本人实时观点或真实发言。

【专家结果】

【{专家 1}｜{本轮任务}】
背景：{一句话背景}
核心判断：{1-2 句，基于赛前情报，不展开成长报告}
他/她会质疑的前提：{1 句}
比分/胜平负倾向：{1 句}

【{专家 2}｜{本轮任务}】
背景：
核心判断：
他/她会质疑的前提：
比分/胜平负倾向：

【Paul-chartroom 会议整理】
共识：
- {最多 2 条}

分歧：
- {最多 1 条}

被排除的信息：
- {最多 1 条}

还缺的信息：
- {最多 1 条}

【章鱼保罗判断】
保罗动作：{一句轻松拟人，例如“保罗先摸了摸平局贝壳，又把触手按在主队小胜上”}
最可能比分：{主队} {比分} {客队}
备选比分：{1-2 个}
胜平负倾向：{一句话}

【娱乐声明】
仅供娱乐，不构成投注建议。
```
